# ADHD Tab Management

一个面向高频浏览场景的 Chrome 扩展：把重要标签留在常驻区，把低频标签收纳到底部，并用访问次数驱动排序。

## 这次调整了什么

- 重做了 popup UI：改成摘要头图 + 操作说明面板 + 两段式标签列表。
- 已移除所有全局快捷键，避免和 Chrome / macOS 系统快捷键冲突。
- 清理了 popup 调试按钮和隐藏的页面调试入口，减少“界面没写、代码还留着”的残留。
- 不再把访问次数写进标签标题，避免浏览器侧边栏出现 `[number]` 这类像异常数据的前缀。

## 功能

- `🔥 常驻标签`
  适合需要频繁切换的工作页。
- `📁 已收纳标签`
  适合临时页、参考页和待关闭标签。
- `访问计数`
  记录当前窗口内每个标签的访问次数，只在 popup 里展示。
- `自动排序`
  常驻区和收纳区都会按访问次数降序排列。
- `批量关闭`
  一键关闭所有已收纳标签。

## 交互方式

当前版本不提供任何全局快捷键，所有操作都通过 popup 完成：

- 点击标签卡片直接切换到对应页
- 点击右侧按钮在常驻 / 收纳之间切换
- 点击底部按钮批量关闭已收纳标签

## Popup UI

新面板分成三层：

- 顶部摘要：显示当前窗口总标签数、常驻数、已收纳数。
- 操作方式面板：说明当前版本的点击式工作流。
- 标签分组：常驻区和收纳区各自独立滚动、独立排序。

每个标签项会显示：

- favicon
- 清理过前缀的标题
- 域名
- 访问次数
- 热度条
- 直接切换优先级的按钮

## 本地加载

1. 打开 `chrome://extensions/`
2. 开启右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前项目目录

## 关键实现

- `manifest.json`
  不再定义全局快捷键命令。
- `background.js`
  负责优先级管理、访问计数、排序、左右切换常驻标签。
- `popup.js`
  负责读取标签状态并渲染 popup 面板。
- `content.js`
  负责更新页面标题和页面内的优先级提示。

## 开发提示

- 修改 `manifest.json` 后需要在扩展页重新加载
- 这是一个纯前端扩展项目，没有构建步骤
